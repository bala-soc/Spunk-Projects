# Splunk SIEM Log Analysis Projects

This repository contains a collection of hands-on projects focused on analyzing different types of log data using **Splunk SIEM**.  

Each project follows a structured, step-by-step approach to help understand log ingestion, field extraction, analysis, anomaly detection, and security monitoring.

These projects are designed to demonstrate **practical SOC Analyst (L1) skills**, including log analysis, threat detection, and incident investigation.

### 1. [Analyzing DNS Logs Using Splunk SIEM](https://github.com/bala-soc/Spunk-Projects/blob/main/1.%20Analyzing%20DNS%20Logs%20Using%20Splunk%20SIEM.md)
This project provides a step-by-step guide for analyzing **DNS (Domain Name System)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading DNS log files into Splunk
- Extracting relevant fields (query, response, source IP, destination IP)
- Analyzing DNS query patterns
- Detecting anomalies such as suspicious or excessive DNS requests
- Monitoring DNS traffic for potential threats (DNS tunneling, malware communication)

---

### 2. [Analyzing FTP Logs Using Splunk SIEM](https://github.com/bala-soc/Spunk-Projects/blob/main/2.%20Analyzing%20FTP%20Logs%20Using%20Splunk%20SIEM.md)
This project guides you through analyzing **FTP (File Transfer Protocol)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading FTP log files into Splunk
- Extracting fields such as username, source IP, commands, and file names
- Analyzing file transfer activity and frequency
- Identifying suspicious login attempts or unauthorized file access
- Monitoring FTP traffic for anomalies

---

### 3. [Analyzing HTTP Logs Using Splunk SIEM](https://github.com/bala-soc/Spunk-Projects/blob/main/3.%20Analyzing%20HTTP%20Logs%20Using%20Splunk%20SIEM.md)
This project outlines the process of analyzing **HTTP (Hypertext Transfer Protocol)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading HTTP log files into Splunk
- Extracting fields such as URL, HTTP method, status code, source IP
- Analyzing HTTP request and response patterns
- Detecting anomalies like repeated failed requests or suspicious URLs
- Monitoring web traffic for security threats

---

### 4. [Analyzing SSH Logs Using Splunk SIEM](https://github.com/bala-soc/Spunk-Projects/blob/main/4.%20Analyzing%20SSH%20Logs%20Using%20Splunk%20SIEM.md)
This project provides a comprehensive guide for analyzing **SSH (Secure Shell)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading SSH log files into Splunk
- Extracting fields such as username, source IP, authentication status
- Analyzing SSH login patterns
- Detecting brute-force attacks or unauthorized access attempts
- Correlating SSH logs with other log sources for better threat detection

---

### 5. Analyzing Tunnel Logs Using Splunk SIEM
This project demonstrates how to analyze **tunnel traffic logs** (e.g., GRE, IPv4, IPv6) generated by **Zeek IDS** using Splunk SIEM.

**Key Focus Areas:**
- Uploading Zeek tunnel log files into Splunk
- Extracting tunneling-related fields
- Analyzing tunnel usage and protocol distribution
- Detecting anomalies such as unexpected tunneling behavior
- Correlating tunnel logs with network and security logs

---

### 6. Analyzing SMTP Logs Using Splunk SIEM
This project provides a structured approach for analyzing **SMTP (Simple Mail Transfer Protocol)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading SMTP log files into Splunk
- Extracting sender, recipient, IP address, and status fields
- Analyzing email traffic patterns
- Detecting anomalies such as spam or phishing attempts
- Monitoring SMTP activity for suspicious behavior

---

### 7. Analyzing DHCP Logs Using Splunk SIEM
This project offers guidance on analyzing **DHCP (Dynamic Host Configuration Protocol)** log files using Splunk SIEM.

**Key Focus Areas:**
- Uploading DHCP log files into Splunk
- Extracting fields related to IP assignment and MAC addresses
- Analyzing IP address allocation patterns
- Detecting anomalies such as IP conflicts or rogue devices
- Monitoring DHCP traffic for network security issues
